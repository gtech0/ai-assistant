**Название проекта:** Доставим.Всё — сервис доставки еды из ресторанов и продуктов из магазинов

**Цель:** Разработать платформу, объединяющую рестораны, продуктовые магазины и курьерские службы для быстрой доставки еды и продуктов конечным пользователям.

**Функциональные требования:**

1. **Мобильное приложение для пользователей (iOS / Android)**
   - Просмотр ресторанов и магазинов поблизости с фильтрацией по кухне, рейтингу, времени доставки.
   - Поиск по названию блюда, продукта или ресторана.
   - Добавление блюд/товаров в корзину, изменение количества, удаление.
   - Оформление заказа с выбором времени доставки, адреса, способа оплаты.
   - Онлайн‑оплата картой, Apple/Google Pay, наличными курьеру.
   - Отслеживание статуса заказа и местоположения курьера на карте в реальном времени.
   - История заказов с повторным заказом.
   - Система отзывов и оценок (для ресторанов и блюд).
   - Push‑уведомления: подтверждение заказа, старт доставки, прибытие курьера, акции.

2. **Веб‑панель для ресторанов / магазинов**
   - Управление меню / ассортиментом (добавление, редактирование, цены, фото).
   - Просмотр входящих заказов в реальном времени.
   - Подтверждение заказа, отметка о готовности.
   - Просмотр истории заказов и выручки.
   - Управление акциями и скидками.

3. **Административная панель (веб)**
   - Модерация ресторанов, магазинов и пользователей.
   - Поиск и блокировка нарушителей.
   - Просмотр метрик: количество заказов, средний чек, время доставки.
   - Настройка комиссии для ресторанов и магазинов.

4. **Система курьеров**
   - Приложение для курьеров (iOS/Android) с приёмом заказов, кнопкой принятия, картой с маршрутом.
   - Загрузка документов (паспорт, водительские права) через админку.
   - Автоматическое распределение заказов с учётом загруженности и геопозиции.

**Нефункциональные требования:**
- Высокая надёжность: система должна быть доступна 99.95% времени.
- Пиковая нагрузка: до 500 заказов в минуту.
- Время ответа API: не более 200 мс для 95% запросов.
- Конфиденциальность: персональные данные пользователей хранятся в зашифрованном виде.
- Геолокация курьеров: частота обновления 5 секунд, точность ±25 м.

**Предполагаемый стек технологий:**
- Бэкенд: Python (Django + DRF), PostgreSQL (основная БД), Redis (кэш, брокер задач), Celery (фоновые задачи).
- Frontend (веб-панели): React + Ant Design.
- Мобильные приложения: Flutter (чтобы один код для iOS и Android).
- Курьерское приложение: Flutter или отдельно нативно (Android).
- Карты: Яндекс Карты SDK для мобильных и Leaflet/OpenStreetMap для веба.
- Инфраструктура: Docker, оркестрация на Kubernetes (для масштабирования), Nginx, Gunicorn + Uvicorn (для WebSocket уведомлений).
- Мониторинг: Prometheus + Grafana.

**Параметры проекта (исходные):**
- Длительность: 9 месяцев (разработка MVP за 6, затем доработки и бета‑тестирование).
- Бюджет: 12 000 000 ₽ (разработка, сервера, лицензии, маркетинг).
- Команда: 8 человек (PM, 2 бэкенд‑разработчика, 2 фронтенд‑разработчика (веб), 2 мобильных разработчика (Flutter), 1 QA‑инженер).
- Уровень квалификации команды: 3 из 5 (средний, но есть опыт со схожими проектами).

**Известные ограничения:**
- Законодательство: требуется обработка персональных данных в соответствии с 152‑ФЗ.
- Платежи: интеграция как минимум с двумя агрегаторами (ЮKassa, CloudPayments) для отказоустойчивости.

**Жизненный цикл:** планируется использовать гибридную модель – Scrum с двухнедельными спринтами, но документация и архитектурная схема ведутся по Waterfall (для отчётности перед инвесторами).
